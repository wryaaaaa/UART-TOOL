# UART 暗黑专业串口助手 — 设计文档

## 概述

基于纯浏览器端（Web Serial API）的专业串口调试助手，单文件 HTML，暗黑高级风 UI，支持 `{变量名:数值}` 格式的传感器数据自动解析和实时趋势图绘制。AI Agent 可通过浏览器自动化工具读取语义化 DOM 数据进行辅助调试。

## 运行环境

- **浏览器**: Chromium 内核（Chrome/Edge），需要 Web Serial API 支持
- **部署方式**: 单文件 `.html`，双击打开或本地 HTTP 服务器
- **外部依赖**: Chart.js 4.x（CDN 加载，jsdelivr，需要翻墙环境）

## 核心模块

### 1. SerialManager — 串口管理

**职责**: 封装 Web Serial API

- `connect(port, options)` — 打开串口，配置波特率/数据位/停止位/校验位/流控
- `disconnect()` — 关闭串口
- `send(data: Uint8Array)` — 发送数据
- `onData(callback)` — 注册数据接收回调
- `listPorts()` — 枚举可用串口
- 热插拔检测: 监听 `navigator.serial` 的 connect/disconnect 事件

### 2. PortPanel — 串口列表 + 设置

**职责**: 串口选择和参数配置 UI

- 可用串口下拉列表 + 刷新按钮
- 波特率预设（9600/19200/38400/57600/115200 + 自定义）
- 数据位(7/8)、停止位(1/2)、校验位(none/even/odd)、流控(none/hardware)
- 连接/断开按钮

### 3. ReceiveView — 数据接收区

**职责**: 原始数据展示

- Hex / ASCII 显示模式切换
- 时间戳显示（精确到毫秒）
- 自动换行开关
- 实时滚动，暂停滚动按钮
- 导出按钮: TXT（纯文本日志）、JSON（结构化）
- 每条数据行的 DOM 结构:
  ```html
  <div class="receive-line" data-timestamp="1717423200000"
       data-hex="7B483A31352C..." data-ascii="{H:15, T:12}">
    <span class="time">14:30:00.123</span>
    <span class="content">{H:15, T:12}</span>
  </div>
  ```
- 右键菜单: "添加到趋势图" → 选中文本传入 VariableParser

### 4. SendPanel — 数据发送区

**职责**: 数据发送

- Hex / ASCII 输入切换
- 单次发送、定时发送（可配置间隔 ms）
- 发送历史（最近 N 条，点击复用）
- 发送快捷键支持（Enter 发送）

### 5. VariableParser — 变量解析引擎

**职责**: 从原始文本中提取结构化变量数据

- 正则: `/\{([^}]+)\}/g` 匹配花括号内容
- 解析 `key:value` 对，value 支持数字/浮点数
- 容错: 格式不匹配的数据静默跳过
- 输出: `[{name: "H", value: 15, timestamp: 1717423200000}, ...]`
- 预配置变量名列表: 用户填入变量名（如 "H, T"），只有匹配的变量才进入趋势图

### 6. TrendChart — 趋势图

**职责**: Chart.js 驱动的实时折线图

- 暗黑主题: 深色背景 + 霓虹色系线条
- 多变量多线: 每条变量一个独立折线，支持动态添加/删除
- 预配置模式: 在趋势图面板添加变量名（如 H → "湿度", T → "温度"），自动追踪
- 右键模式: 从接收区选中 `{H:15, T:12}` → 右键 → 全部或选择变量加入
- X 轴: 时间，Y 轴: 数值
- 支持缩放、平移（Chart.js zoom plugin 或原生配置）
- 数据点上限: 最近 1000 个点，超出自动滚动
- 图表面板可折叠
- 导出 JSON: 趋势图结构化数据导出

### 7. LayoutManager — 标签页/分屏管理

**职责**: 管理多串口连接的视图布局

- 默认标签页模式: 每个串口连接一个标签页
- 点击标签切换当前显示的串口
- 拖拽标签到右侧 → 分屏模式（左右并排）
- 分屏模式下可拖回合并
- 每个标签页关联独立的 ReceiveView + SendPanel + TrendChart

## 数据流

```
串口硬件
    │
    ▼
SerialManager (ReadableStream 读取)
    │  原始字节流 (Uint8Array)
    ├────────────────────────────────────┐
    ▼                                    ▼
ReceiveView                         VariableParser
(原始数据展示)                       (正则提取 {key:value})
    │                                    │
    │                                    ▼
    │                               TrendChart
    │                               (结构化数据 → 折线图)
    │
    ▼
AI Agent 可读 DOM
(data-timestamp / data-hex / data-ascii 属性)
```

- 两条路径并行，互不阻塞
- 原始数据始终显示在 ReceiveView
- 只有匹配格式的数据才进入趋势图

## AI Agent 可读性设计

| 数据 | 访问方式 |
|------|----------|
| 接收区最新 N 条数据 | `document.querySelectorAll('.receive-line[data-ascii]')` |
| 指定变量最近数据点 | `document.querySelector('.trend-data-json[data-varname="H"]')` |
| 串口连接状态 | `document.querySelector('[data-port-status]').dataset.portStatus` |
| 收发字节计数 | `document.querySelector('[data-stats]').dataset` |
| 趋势图最新值 | 隐藏 DOM: `<div class="trend-latest" data-varname="H" data-value="15.3" data-time="...">` |

每个状态变化都反映在 DOM 属性上，AI Agent 无需执行 JS，只需读取 DOM。

## 错误处理

| 场景 | 处理方式 |
|------|----------|
| 串口意外断开 | 状态栏变红 + 通知提示，自动停止读取，保留已收数据 |
| 非法数据格式 | VariableParser 静默跳过，不影响接收区显示 |
| 发送失败 | 发送区输入框红色边框闪烁 1 秒 |
| Chart.js CDN 加载失败 | 趋势图区域显示 "图表库加载失败" + 重试按钮 |
| 浏览器不支持 Web Serial | 顶部横幅提示 "请使用 Chrome/Edge 打开" |
| 用户拒绝串口权限 | 提示 "需要授予串口访问权限" |

## 数据保存与回放

- **接收区**: 支持导出 `.txt`（纯文本日志 + Hex dump）和 `.json`（带时间戳的结构化数组）
- **趋势图**: 支持导出 `.json`（解析后的变量时间序列数据）
- 导出文件命名: `uart-log-YYYYMMDD-HHmmss.txt` / `uart-log-YYYYMMDD-HHmmss.json`

## 技术栈

| 层 | 技术 |
|----|------|
| 结构 | HTML5（单文件） |
| 样式 | CSS3，CSS 自定义属性（暗黑主题变量系统） |
| 逻辑 | 原生 JavaScript (ES6+)，无框架 |
| 图表 | Chart.js 4.x（CDN: jsdelivr） |
| 串口 | Web Serial API |
| 字体 | 系统等宽字体栈 |

## 颜色主题（暗黑高级风）

- 背景: `#0d1117` ~ `#161b22`
- 面板: `#1c2128`，边框 `#30363d`
- 文字: `#c9d1d9`（主）、`#8b949e`（次要）
- 强调色: `#58a6ff`（蓝）、`#3fb950`（绿，连接状态）
- 错误色: `#f85149`（红，断开/错误）
- 图表线色: `#58a6ff` `#3fb950` `#d2a8ff` `#f0883e` `#f85149`（霓虹色系）
- 字体: `'Segoe UI', system-ui, -apple-system, sans-serif`
- 等宽字体: `'Cascadia Code', 'JetBrains Mono', 'Consolas', monospace`

## 实现顺序

1. **骨架搭建**: HTML 布局 + CSS 暗黑主题 + 响应式基础
2. **SerialManager**: Web Serial API 读写封装
3. **ReceiveView**: 数据接收展示 + 格式切换
4. **SendPanel**: 发送功能 + 定时发送 + 历史
5. **VariableParser**: 解析引擎
6. **TrendChart**: Chart.js 集成 + 实时多线图
7. **LayoutManager**: 标签页/分屏拖拽
8. **导出功能**: TXT/JSON 导出
9. **精细打磨**: 动画、键盘快捷键、错误边界、AI 可读性数据属性
